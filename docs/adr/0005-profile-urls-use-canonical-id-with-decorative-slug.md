# Profile URLs use a canonical short-ID with a decorative slug

Practitioner profile URLs follow the pattern `/p/<short_id>/<auto_slug>`, where `short_id` (8-10 char base62, distinct from the database primary key) is canonical and `auto_slug` (kebab-cased `firstname-lastname-practice-town`) is auto-generated and regenerated whenever those fields change. Visiting the URL with a stale or arbitrary slug — `/p/<short_id>/anything` — issues a 301 redirect to the current canonical slug; the slug is never consulted for routing, only for display and SEO. This shape exists because names change (married names, deed polls), practices move, and an immutable canonical ID is the only way to keep links, bookmarks, and search rankings stable across those changes; the slug is fully system-generated to avoid handle-squatting and moderation overhead in Phase 1.
