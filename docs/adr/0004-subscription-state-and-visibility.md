# Subscription state interacts with visibility via Stripe's dunning window

A Practitioner is visible to consumers only when `verification_status = 'verified'` AND `subscription_status IN ('active', 'past_due', 'trialing')`. Keeping `past_due` Practitioners visible during Stripe Smart Retries' ~3-week dunning window is deliberate — a card failure is usually a card problem, not a customer trying to leave, and cutting them off the moment a renewal fails punishes someone who is actively trying to pay. Cancellations are honoured until period end (Stripe's `cancel_at_period_end`); profiles are row-preserved-but-hidden on `canceled`/`unpaid` so that resubscription restores the same profile rather than forcing a rebuild; a `revoked` verification (Practitioner struck off the regulator's register) triggers immediate Stripe cancellation with a prorated refund.
