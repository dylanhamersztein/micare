# Verified-only visibility with synchronous regulator check at signup

A Practitioner is visible to consumers only with `Verification Status = verified`, established by a synchronous scrape of the regulator's public register during signup (10s timeout); re-verification runs as a weekly cron, with a `last_verified_at` timestamp plus alerting standing in for an explicit `lapsed` state in Phase 1. This couples signup uptime to the regulator's website but preserves the invariant *every visible profile is currently verified* — the product's central trust claim. The `pending` status exists only as a fallback for the rare sync-timeout case and is not consumer-visible; ops handles those manually.
